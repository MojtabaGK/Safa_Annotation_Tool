# Contributing to Safa Annotation Tool

First off, thank you for considering contributing to Safa Annotation Tool! 🎉

## How Can I Contribute?

## 📝 Citing This Project

If you use Safa Annotation Tool in your research, project, or publication, please cite it as:

```
Mojtaba GK. (2024). Safa Annotation Tool: A Simple Image Annotation Tool. 
GitHub. https://github.com/MojtabaGK/Safa_Annotation_Tool
```

### 🐛 Reporting Bugs
Found a bug? Please [open an issue](https://github.com/MojtabaGK/Safa_Annotation_Tool/issues) with:
- Clear description of the problem
- Steps to reproduce
- Screenshots if possible
- Your OS and Python version

### 💡 Suggesting Features
Have an idea? Open an issue with:
- Feature description
- Why it would be useful
- Any implementation ideas you have

### 🔧 Fixing Issues
Check the [issues](https://github.com/MojtabaGK/Safa_Annotation_Tool/issues) page:
- Look for issues labeled `good-first-issue` or `help-wanted`
- Comment on the issue that you're working on it
- Submit a Pull Request with your fix

### 📚 Improving Documentation
Help make the documentation better:
- Fix typos or unclear sections
- Add examples or usage tips
- Improve the README or wiki

### 🖼️ Adding Sample Data
Contribute sample images or annotation examples for testing.

### 🌐 Translations
Help translate the tool's interface or documentation to other languages.

## Development Setup

```bash
# Clone the repository
git clone https://github.com/MojtabaGK/Safa_Annotation_Tool.git
cd Safa_Annotation_Tool

# Install dependencies
pip install -r requirements.txt

# Run the application
python main.py
```

---

## Pull Request Process

1. **Fork** the repository
2. **Create a branch** for your changes
3. **Make your changes** and test them
4. **Commit** with a clear message
5. **Push** to your fork
6. **Open a Pull Request** with description of changes

**Commit Message Format:**
```
type: short description

- feat: new feature
- fix: bug fix
- docs: documentation
- refactor: code improvements
- style: formatting
```

---

## Code Style

- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/)
- Use clear variable/function names
- Add comments for complex code
- Keep functions focused and short
